# A source-rooted C5 contradiction exits via a Routed fix-the-Source flow, surfaced through a two-view Reconcile modal, backed by a destination-aware in-place source overwrite

[ADR-0034](0034-c5-contradiction-only-retire-similarity-buckets.md) narrowed C5 to same-question contradictions and claimed the check was **"convergent by construction"** — every finding a contradiction Reconcile can fix, so a fixed finding disappears. Live-console testing disproved that claim for one class. C5 flagged `退款流程 / 退貨期限提醒` (and, at root, `退款申請窗口 / 退貨期限提醒`); opening **Reconcile fails the grounding re-check** and *no* wiki-layer edit — not even hand-editing both drafts to identical text — can make it pass.

**Confirmed root cause.** The two pages are each **faithfully grounded** in their own Sources, but the Sources themselves disagree: `docs/demo-zh/退款與退貨.md#退款申請窗口` says **14 天**, the planted contradiction carrier `docs/planted-zh/退貨期限提醒.md` says **30 天** (corpus_v2 §P designed finding). `reconcile._collect_union_sections` grounds the reconcile draft against the union of both pages' whole Source files, so the union contains BOTH the 14-day and 30-day sections; facing a self-contradictory union, the verifier cannot support any single "N 天" claim, so `apply` always fails. The contradiction lives in the **Source layer**, and the curated wiki layer structurally cannot repair it — Reconcile ("make the two pages consistent against their Sources' union") is the wrong tool when the union itself is inconsistent.

## Decision

1. **C5 gains a second remediation class: `secondary_route="fix-source"`, mirroring C3 exactly** ([ADR-0029](0029-failed-grounding-quarantine-routed-source-fix.md)). Its primary tier stays `"authored"` (Reconcile, unchanged for wiki-rooted findings); it ALSO carries the Routed navigation for the **source-rooted** case. A C5 finding is source-rooted when its Reconcile draft cannot be grounded against the union of Sources because that union self-contradicts — the same "the missing ingredient is knowledge only the human can supply" definition that makes C3 Routed.

2. **Detection is lazy, at Reconcile generate time; the human judges, the system never auto-classifies.** `generate_reconcile` already grounds the draft against the union — a grounding **failure** is the source-rooted signal. But a single failure is not a proof (the LLM may have missed a legitimate *conditional* reconciliation — e.g. a Source that consistently says "14 天 non-perishable, 7 天 perishable" is wiki-rooted, not source-rooted), so the outcome is presented to the curator rather than acted on. This deliberately avoids the false shortcut "both pages live + judged `direct` ⟹ Sources conflict" — true only when the union is *unconditionally* inconsistent, which only the reconcile-generate grounding attempt actually tests.

3. **A two-view Reconcile modal lets the curator see which layer to fix.** The modal header toggles between:
   - **Wiki comparison** (existing) — the two pages + the editable reconcile draft + grounding report → the wiki-rooted exit (edit → apply, Authored Reconcile).
   - **Source comparison** (new) — the two pages' **cited Source sections** side by side (anchored, e.g. `退款與退貨.md#退款申請窗口` = 14 天 ‖ `退貨期限提醒.md#退貨期限提醒` = 30 天), each with a `/read/file` view link → the source-rooted exit.

   The default view follows the generate outcome: grounding **pass** → Wiki comparison; grounding **fail** → Source comparison (with "the Sources disagree; this cannot be auto-reconciled at the wiki layer"). Both toggles are always present. This maps onto the project's two axes — Wiki view is page-vs-page (Coherence); Source view drops to the page-vs-Source layer to see the root.

4. **fix-source reuses ADR-0029's Routed machinery**, with one difference: C3 is "one Source cannot support a claim"; C5 is "**two** Sources disagree — the human edits one to match the other," so the Source view presents both files with edit entry points. No in-app Source editor (ADR-0003/0029): the human edits locally and Uploads the corrected byte-stream.

5. **Batch, not one-at-a-time.** fix-source accumulates targets across findings into a persistent panel (the coverage-fill banner pattern, generalised to a multi-target checklist); the curator corrects all flagged Sources, then **one batch Upload → one force re-ingest → one deep-audit** clears the resolved findings together. A per-finding "fix one, re-audit, fix next" stepper is rejected as hostile UX.

6. **Destination-aware in-place Source overwrite** (closes a loop-breaking gap that also affected C3). `upload._upload_one_file` writes every `.md` to `docs/` **root**, ignoring the Source's origin subdirectory; corpus_v2 Sources live under `docs/demo-zh/`, `docs/planted-zh/`, so a same-name re-upload lands a *second* file at the root, and the next re-ingest fails `ambiguous_source` (`docs_dir.glob("**/{basename}")` matches 2+ files — `ingest._resolve_single_source_pairs` / `lint._resolve_c3_source_path`). The fix-source Upload instead overwrites the Source's **resolved origin relpath** in place. Guard: overwrite an EXISTING Source under `docs/` only — never create new nesting, never traverse; if the origin path cannot be uniquely resolved (`ambiguous`/`missing`), refuse with a clear message rather than fall back to root.

7. **The grounding union stays whole-file** (rejected: narrowing to cited sections). Whole-file is load-bearing for detection: for `退款流程 / 退貨期限提醒`, the whole-file union pulls in the sibling 14-day section, correctly flagging the pair source-rooted; a cited-section union would omit it, letting a locally-grounded-but-globally-wrong "30 天" be written into `退款流程` (which its own Source file says is 14 天) — moving the contradiction, not fixing it.

8. **The `tension` bucket survives** (ADR-0034) but its un-clearable subset now flows to fix-source like `direct` does — a `tension` whose omitted fact lives in a contradicting Source is source-rooted too.

## Consequences

- **ADR-0034's "convergent by construction" is corrected, not abandoned.** C5 is convergent with **two exits**: wiki-rooted findings converge under Reconcile (fixed → re-judge `none`); source-rooted findings converge under fix-source (edit the Source so the union agrees → re-ingest → re-lint `none`). No finding class is a dead end — which is the property ADR-0034 promised and this ADR actually delivers. See the amendment appended to ADR-0034.
- **The whole fix-source loop is verified end-to-end against the real corpus** before this is called done (the operator's stated acceptance bar): edit `退貨期限提醒.md` 30→14 → force re-ingest → the page regenerates as 14 天 → deep-audit re-judges the pair `none`. The C5 verdict cache cannot hide the fix (keyed on both pages' body hashes, #446/#473); a re-ingested page busts the key and re-judges live.
- **Recall is a separate concern.** The genuine `direct` pair `退款申請窗口 / 退貨期限提醒` is not an F1 candidate (different Source files) and depends on F3 (BM25) to surface. Whether it is currently reported is verified empirically during implementation; if F3 misses it, a candidate-selection fix (cross-Source same-fact pairing) is filed as its own issue, not folded here.
- **Source rename/removal is explicitly out of scope and deferred to its own decision** — a deployed operator cannot yet rename or retire a mistyped/duplicate Source. fix-source is locked to *content overwrite, same filename*, which structurally avoids the "upload under a new name, old file stuck forever" trap. The governed Source-lifecycle surface (supersede / rename-with-alias-redirect / soft-retire) is a follow-up grill that will supersede ADR-0029's "removal only out-of-app" clause.
- **Invariants (tagged for the reviewer, per CODING_STANDARD §2.5).** **Invariant** — a C5 finding always has an executable exit: Reconcile (wiki-rooted) or fix-source (source-rooted). **Invariant** — the system never auto-classifies source-vs-wiki root; the curator chooses via the two-view modal. **Invariant** — the fix-source Upload overwrites only an existing Source at its resolved origin path under `docs/`, and refuses (never writes) when that path is `ambiguous`/`missing`. **Invariant** — the reconcile grounding union is the whole cited Source files, unchanged.

## Amendment (2026-07-20, [ADR-0043](0043-c5-in-modal-source-editor-staged-batch-byte-source.md))

**Decision 4's "No in-app Source editor" is superseded** (issue #632, operator report): the silent accumulate-behind-the-modal flow was illegible, and the local-edit-and-reupload round-trip hostile for one-line corrections. "Fix this Source" now opens an **in-modal whole-file editor**; "Stage correction" holds the corrected text client-side; staged corrections upload through the **same batch pipeline** (decision 5's one-batch-Upload → one-force-re-ingest → one-deep-audit shape and decision 6's server-side overwrite guards are unchanged — the editor is only an alternate byte-source for the same `/upload` ingress, still human-authored). Decision 3's "the edit entry point never fetches anything itself" wording is superseded by the editor's read-only `GET /read/file` load; the no-mutation intent stands. The batch panel and file-picker path survive as the multi-finding / local-editing route.

## Amendment (2026-07-07, [ADR-0038](0038-c5-source-rooted-detected-by-convergence-rejudge.md))

**Decision 2's detection signal is wrong and is superseded.** This ADR made a reconcile **grounding failure** the source-rooted signal, on the §7 premise that "facing a self-contradictory union, the verifier cannot support any single 'N 天' claim, so `apply` always fails." Live-console testing (issue #545) disproved it: grounding is an *existence* check, so a self-contradictory whole-file union **individually supports each** faithful draft (14 天 from one Source, 30 天 from the other) — a still-contradicting A=14/B=30 reconcile draft passes grounding, and (grounding being unseeded) the outcome even flaps run-to-run. [ADR-0038](0038-c5-source-rooted-detected-by-convergence-rejudge.md) replaces the signal: source-rooted is detected by **re-judging the reconcile drafts** with the C5 contradiction oracle (`_judge_page_pair`) for cross-page convergence, and grounding is demoted to a faithfulness-only check. Everything else in this ADR stands — the fix-source Routed exit, the two-view modal, the destination-aware in-place overwrite, and the whole-file grounding union (retained: still correct for faithfulness and for detection recall). Only *what decides the default view and enables Apply* changes: it now keys on a new `converged` field, not on `grounding.passed`.
