# C5 narrowed to contradiction-only; the `tension`/`duplicate` similarity buckets retired

C5 (the Coherence [[lint-axis]] contradiction check) was built as a *similarity* judge: a candidate filter (F1 shared-source ∪ F3 BM25/Jaccard) fed an LLM that returned one of four severities — `direct` (real conflict), `tension` (*"same topic, scope or wording differences that could confuse readers"*), `duplicate` (same concept, no conflict), `none` — and every finding got the one Reconcile action. On the real demo corpus this produced **22 findings, of which only 1 was a genuine contradiction**: 9 `tension` were adjacent-but-distinct false positives (cancel vs pause, pickup-scope vs item-restrictions, pickup-fee vs home-delivery-fee), and 12 `duplicate` were consistent redundancy. Worse, the noise was **un-clearable**: Reconcile's contract is "resolve a disagreement," so on a pair that does not disagree it is a no-op — the finding re-fires on every audit (verified: only the 1 `direct` disappears after Reconcile). The check was, in the user's words, "看似嚴格但毫無幫助" — and the data agreed (95% noise).

**Decision.** C5 detects only whether two pages give a reader **incompatible answers to the same question**. The judge prompt is rewritten to make `none` the default and to name the false-positive classes explicitly (different-but-adjacent actions, broader-vs-specific, one page merely more detailed, and *consistent* redundant coverage all → `none`). `severity` drops to `Literal["direct", "tension", "none"]` — the `duplicate` value is **retired**. `tension` survives but is re-scoped to the charter's real case (same question, one page cherry-picks/omits a condition the other states, so reading it alone misleads), which Reconcile *does* resolve.

This makes C5 **convergent by construction**: the only findings it emits are contradictions Reconcile can fix, so a fixed finding disappears (re-judge → `none`) and a non-issue is never raised.

**Verified on the real corpus (live re-judge, `gpt-4o-mini`):** the narrowing took the audit from **22 findings (1 direct / 9 tension / 12 duplicate)** to **4 (1 direct / 3 tension / 0 duplicate)**. Every named false positive dropped to `none` (cancel/pause, 取消訂閱/暫停訂閱, and all 12 consistent-redundancy duplicates); the one genuine contradiction (`搭配貨到付款 / 貨到付款`) survived as `direct`; the 3 surviving `tension`s are all the charter's cherry-picked-omission case (e.g. `退款流程 / 退貨期限提醒`, where one page omits the 30-day window the other states), each Reconcile-actionable. The single Reconcile action is therefore no longer a mismatch — there is no verdict left that it cannot resolve. This aligns the check with its CONTEXT.md charter (Coherence = genuine page-vs-page contradiction, the axis [[grounding-check]] structurally cannot see); `tension`-as-similarity and `duplicate` were scope-creep beyond it.

**Redundancy is deliberately out of C5's scope.** Consistent duplication is harmless — and if two pages ever state a fact *differently*, C5 still catches it as `direct`, so the dangerous case (divergent duplication) is preserved. Genuine duplicates that share a slug base (ingest's `-2`/`-3` collisions, e.g. `會員日優惠` / `會員日優惠-2`) remain **C4**'s job, which already ships Merge and Differentiate ([[adr-0028]]). Cross-name semantic-duplicate detection (the old `duplicate` bucket's "absorbs C4-b" ambition) is dropped.

## Considered Options

- **Add a Merge action + a Dismiss to service all three buckets (rejected).** Treats symptoms: keeps the similarity-based over-firing and bolts a second action onto C5 for redundancy it was never chartered to detect. More surface area, still low precision, and Dismiss becomes a chore the curator runs 21 times.
- **Add only a persisted Dismiss (rejected as the primary fix).** Lets a curator silence a false positive without a destructive Reconcile, but leaves 95% of output as noise to hand-dismiss one by one; the audit stays useless until fully dismissed. Kept as a *deferred residual valve* (below), not the fix.
- **Claim-level structured contradiction detection (deferred, named as the ceiling).** Reuse the grounding claim-extractor, align same-subject / opposite-polarity claim pairs instead of asking an LLM "do these blobs feel similar." Highest precision, but significant engineering; prompt-narrowing captures most of the benefit for a fraction of the cost. The honest next step, not gold-plated now.

## Consequences

- `PagePairFinding.severity` is `Literal["direct", "tension", "none"]`; the Markdown report and the Console render only `direct`/`tension`. `with_structured_output(PagePairFinding)` can no longer emit `duplicate`, so the retirement is enforced at the schema, not by a filter.
- The candidate filter (F1 ∪ F3 similarity) is **unchanged** — it is only a cheap cost gate; precision now lives entirely in the judge, whose default is `none`.
- The prompt is part of the C5 verdict-cache **generation salt** (`_c5_generation_salt`, [[adr]]-none / issue #473), so this change invalidates every cached verdict — the next audit re-judges live rather than serving stale similarity verdicts. Correct and intended.
- **Invariant** (tagged for the reviewer, CODING_STANDARD §2.5) — C5 surfaces only contradictions Reconcile can converge: a C5 finding disappears **iff** the content is fixed so the re-judge returns `none`. No finding class that Reconcile cannot resolve is ever reported.
- **Deferred residual Dismiss.** If real over-calls survive the narrowed judge, the exit is a Dismiss mirroring C4's differentiate-exemption sentinel ([[lint]] `_differentiated_group`) — keyed on the pair + both bodies' content hashes, persisted in a *committed* sidecar (not the gitignored `.kb/` verdict cache), re-opening automatically when either page's body changes so a stale dismissal can never hide a newly-introduced contradiction. Built only when the false-positive rate justifies it, per scope discipline.
- The `會員日優惠 / 會員日優惠-2` recurrence is **not** a C5 concern (both slug on the same base → a C4 collision) and its *root* is at ingest (the `concept` template split one Source Section into two same-titled pages); narrowing C5 removes only the phantom `會員日優惠-2 / 點數折抵` `duplicate`, and the real collision stays with C4.
