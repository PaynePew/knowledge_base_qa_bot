# Aliases live in target-page frontmatter and feed one link-layer resolver (C2, linkify, inbound-refs); assigning one is a human-surface Direct operation; retrieval is untouched

The C2 red-link loop cannot always close: ingest mints slugs from the Source outline freely ([[adr-0027]] deliberately added no slug-hint), so a corpus that mentions `[[PayPal]]` may gain its coverage under a page ingest named `替代付款方式` — the link stays red forever and the honest-miss report can only end in prose ("renaming the link or the page is a curator judgment, done by hand"). The same 2026-07-03 grill also took up wikilink navigation: correctly-resolving `[[links]]` today serve only machines (C2's resolution signal, the C4 merge guard's inbound-reference scan); no surface renders them clickable for a person. Both problems, plus the guard's correctness, want the same missing object: a curator-owned mapping from alternate names to canonical pages, consulted by one shared resolver.

## Decision

1. **Storage: an `aliases:` list in the target page's frontmatter.** The mapping lives on the page it resolves to — correct by construction (a deleted page takes its aliases with it; no dangling table rows), aligned with the Obsidian field the pattern-source vault convention already recognises ([[adr-0003]]), and visible in one place via the `wiki/index.md` projection (which lists aliases beside each page). Because `/ingest` rewrites pages wholesale, `wiki_writer`'s preserve-on-overwrite mechanism extends from `{created}` to `{created, aliases}`; `aliases` is thereby a curator-owned field the sentinel comment's "manual edits will be overwritten" warning explicitly excepts. (Honest scope note: Obsidian itself does not resolve a bare `[[alias]]` to the aliased page — this project's resolution semantics are deliberately stronger; the alignment is field name and location, not behavior parity.)
2. **One resolver, three consumers, link-layer only.** A single resolution facility (home: `slugs.py`) builds the map *existing slugs ∪ aliases → canonical slug*. Its consumers: **C2** (a wikilink is red iff it resolves by neither slug nor alias — the closure the red-link loop was missing), **linkify** (decision 5), and **`find_inbound_references`** (an alias-mediated `[[link]]` is a real inbound reference; the C4 merge guard missing it would approve deletions that break links). Retrieval is deliberately untouched in v1: aliases never enter BM25 tokens or the dense arm — alias-as-search-synonym is a retrieval-behavior change that needs paraphrase-eval evidence and is deferred on its own.
3. **Maintenance: `POST /pages/{slug}/aliases`, Direct class, human surfaces only.** Assigning an alias is reversible, involves no LLM and no synthesis, and the human authors the mapping in the very gesture — Direct by [[adr-0024]]'s seam. It never batches (each mapping is an independent semantic judgment). Two entry points share the endpoint: an **Assign-Alias picker on the C2 card** (the red link's second exit, beside the Routed Fill-via-Import) and a **one-click offer on the honest-miss report** ("make `[[PayPal]]` an alias of the just-minted `替代付款方式`?") — upgrading [[adr-0027]]'s prose ending into an executable close without touching ingest's contract. The CLI gains `kb alias add`. **MCP sees aliases and writes none**: a bare-slug↔page mapping is a semantic assertion about the corpus, and [[adr-0026]]'s posture (each surface gets what its operator's hands can honestly do; agent-as-curator waits for Phase 12) applies even though the operation is Direct-class. Removal in v1 is a frontmatter hand-edit (curator field, preserve-protected).
4. **A new Coherence-axis lint check, C12 alias-collision**, catches the two conflict shapes frontmatter storage admits: an alias colliding with an existing page slug, and two pages claiming the same alias. Advisory with a Direct fix (edit the frontmatter); resolution behavior under collision is deterministic (real slug wins over alias; tie broken lexicographically) so the corpus is never un-queryable while the finding stands. The ID skips to C12 — C7 was never assigned and is left unused.
5. **Linkify, with red styling scoped by surface role.** `[[wikilinks]]` render as navigation wherever page bodies are shown: resolvable (slug or alias) → a link to the canonical page; unresolvable → **red-link styling on the curator surface** (Console `/read/file` viewer — visually the same judgment C2 reports) but **plain-text degrade on the reader surface** (chat answers and the chat-side citation viewer — a reader can fix nothing, so red is noise). Chat answer bodies apply the same rule to any `[[...]]` the LLM echoes from context. The `/read/file` contract is unchanged (raw content; rendering is the client's job); clients consume a new read-only resolution-map endpoint.

## Considered Options

- **A central `wiki/redirects.md` table (rejected).** Single-glance auditability, but rows dangle when their target page is deleted (needs its own hygiene check), the format is bespoke, and it forfeits the Obsidian-field alignment; the see-everything need is met by the `wiki/index.md` projection instead.
- **A `.kb/` mapping (rejected).** `.kb/` holds derived artifacts; putting hand-authored authoritative state there is a category error.
- **Frontmatter without extending preserve-on-overwrite (not viable).** Re-ingest rewrites pages wholesale; aliases would be wiped on every re-sync. The `created` precedent makes the extension mechanical.
- **MCP write parity for aliases (rejected).** An agent autonomously asserting "X names Y" is unsupervised semantic mutation of the resolution layer — the same reasoning [[adr-0026]] used to keep gate-resolving verbs off MCP, applied to a Direct-class op whose *content* is a judgment.
- **Aliases affect retrieval in v1 (rejected).** Changes ranking behavior on top of freshly-tuned Phase 13 hybrid retrieval with no eval evidence; deferred until it can carry a paraphrase-comparison run.
- **Red-link styling on all surfaces (rejected).** Semantically purest, but shows readers a curator signal they cannot act on.
- **A slug-hint to ingest instead of aliases (rejected, re-affirmed).** [[adr-0027]] already rejected coupling lint's finding shape into ingest's outline contract; aliases solve the mismatch *after* ingest without touching it.

## Consequences

- [[adr-0027]] is amended, not reopened: no-slug-hint and the ingest contract stand; the honest-miss report's prose ending becomes an executable assign-alias offer.
- The C4 merge guard grows a natural future move this ADR notes but does not build: on merge-delete, recording the dead variant's slug as an alias of the base (classic redirect-on-merge) would let the reference guard relax. That is a change to [[adr-0028]]'s guard semantics and needs its own decision.
- New-endpoint ledger: `POST /pages/{slug}/aliases` plus one read-only resolution-map endpoint.
- `_REMEDIATION_TAXONOMY` renders C2 with two exits (Routed fill, Direct assign-alias) across Browser/CLI/MCP per [[adr-0017]] — with the MCP rendering of assign-alias as visibility only.
- **Invariants (tagged for the reviewer, per CODING_STANDARD §2.5).** **Invariant** — alias resolution is link-layer only: no alias value enters the Section Index (BM25) or the dense arm in v1. **Invariant** — every consumer of wikilink resolution (C2, linkify, inbound-reference computation) uses the shared resolver; no surface builds its own slug set. **Invariant** — no MCP tool writes aliases. **Invariant** — `/ingest`'s page overwrite preserves the `aliases` frontmatter field alongside `created`. **Invariant** — assign-alias never batches.
