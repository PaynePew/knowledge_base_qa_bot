# Lint remediation is a separate, two-tier operation (Direct vs Authored); lint stays read-only, and Authored fixes go through validated write-back

> **Status:** Amended by [ADR-0024](0024-gated-remediation-human-gate-seam.md) — the governance seam is re-cut from "produces new synthesis?" to "requires a human gate?": Direct vs **Gated**, with Authored and a new **Confirmed** flavour (C11 orphan-delete) under Gated. Everything else here — read-only lint, batch-is-Direct-only, validated write-back, the tier-A scope — remains in force; "tier B" now means the Gated class.

The Lint Pass (Phase 5) surfaces ten findings across four [[lint-axis]]es, and each finding already carries an advisory `suggested_action` *string* — but, except for the C8 Curation Queue (Promote/Discard on Filed Answers), no finding has an *executable* fix. A 2026-07-01 grill framed this as the interviewer's "然後呢? (and then?)": the demo **diagnoses** the wiki's health but cannot **remediate** it, so it is not a finished product. This ADR records the decision to add a [[remediation]] layer, and — because the author must defend the shape in an interview — why remediation is split into two governance tiers rather than one uniform "fix" button.

The organising insight is that every lint finding is a **divergence between two things that should agree**, and the divergence's [[lint-axis]] dictates the fix. This yields a clean two-tier split by a single question — *does the fix produce new curated synthesis?*

- **Direct Remediation (tier A) — no new synthesis.** The fix either re-runs deterministic machinery from an immutable [[source]] ([[ingest]] re-sync: Freshness C6 stale, C3 failed-grounding) or flips a reversible [[filed-answer]] lifecycle bit (Lifecycle C8 promote, C10 discard). The result is deterministic and re-derivable, so it is one-click and needs no review, and the endpoints already exist (`POST /ingest`, `POST /qa/{slug}/promote`, `DELETE /qa/{slug}`). Tier A ships first with **zero new endpoints**.
- **Authored Remediation (tier B) — new synthesis.** The fix writes new curated content: *reconcile/merge* two diverging pages (Coherence C5 contradiction, C4 collision) or *fill* a gap (Coverage C1 coverage-gap, C2 red-link). Here the LLM **drafts** and a curator **approves** before the write — the identical validated-write-back governance the Filed Answer loop already uses ([[adr-0020]]): grounding verification + attribution + human gate.

Two invariants hold across both tiers. **(1) Lint stays read-only.** A Remediation is always a separate operation triggered *from* the report, never folded into `run_lint()`; the report remains a read-only projection (preserving the PRD #65 Q3 / #78 invariant that lint writes only `lint-report.md`). **(2) Authored Remediation never bypasses grounding + human approval** — the same reason [[adr-0020]] rejected auto-publish. Findings are additionally grouped and labelled by [[lint-axis]] identically across the Browser, CLI, and MCP from a **single domain taxonomy** ([[adr-0017]] interface parity), which also carries the zh/en labels, so adding a language or a new check touches one place.

## Considered Options

- **Let `/lint` auto-fix as part of the pass (rejected).** Folds mutation into the read-only pass and, for the Authored cases, auto-writes synthesis into the curated corpus with no human gate — the corpus-pollution / hallucination-accumulation risk [[adr-0020]] already rejected as "auto-publish". Read-only lint + separate remediation keeps the diagnosis trustworthy and the mutation deliberate.
- **Stay advisory-only — findings are hints, the operator edits files by hand (rejected; this is the status quo the grill is fixing).** This *is* the "然後呢" gap. Telling a Browser or CLI user to go hand-edit files on disk defeats the point of the Operator Console — the same reasoning [[adr-0012]] used to justify the qa Discard button.
- **One uniform remediation mechanism for all findings (rejected).** The four axes genuinely need different operations: re-sync (deterministic, re-derived from a Source) is a categorically different act from reconcile/fill (authored, new synthesis). Collapsing them into one mechanism either over-gates the safe deterministic fixes with needless review friction, or under-gates the authored fixes into ungoverned corpus writes. The Direct/Authored line is the natural governance seam.
- **Let the LLM author Coverage pages from nothing (rejected).** C1 coverage-gap and C2 red-link findings have **no [[source]]** — the whole point is that the knowledge is absent. Letting the LLM synthesize a page anyway would produce an ungroundable page that fails [[grounding-check]] by construction. Coverage remediation therefore routes through the existing Import → Ingest pipeline (supply a Source, then ingest), pre-seeded by the finding — not an LLM-from-nothing authoring engine.

## Consequences

- **Tier A first, zero new endpoints.** Console renders per-finding buttons: C6/C3 → `Re-ingest` (`POST /ingest`), C10 → `Discard` (`DELETE /qa`). C8 keeps its **own** Curation Queue block (Promote/Discard) to emphasise the Filed Answer's business weight (deliberate, not an oversight). After any remediation the report re-runs lint with `include_c5=false` (fast, LLM-free, matching the existing Queue Refresh) so the finding visibly disappears and the loop closes on screen.
- **Batch remediation is a Direct-tier-only affordance.** Because Direct Remediations are deterministic and review-free, a section may offer a one-click *batch*: the Freshness axis header re-ingests C6 ∪ C3 in a **single `POST /ingest {sources:[…], force:true}`** call (N → 1). `force` is **mandatory** here — C3 failed-grounding sources are frequently byte-unchanged (grounding failed, the Source did not change), so without `force:true` the hash-skip idempotency (#93) no-ops the retry into a *false fix*; C11 orphans are excluded from the batch (no Source to re-ingest). **Lifecycle stays check-level only** (C8 Promote-all, C10 Discard-all) with **no axis-level button**, because promoting drafts and deleting invalid pages are different intents that must not fire from one click — and a Promote-all must flip all statuses then reindex **once**, not trigger N full BM25 rebuilds (a batch-promote endpoint, deferred). **Authored-tier findings (Coherence/Coverage) get no batch at all**: bulk-approving LLM-drafted merges/fills would defeat the per-item human approval gate that *defines* the tier. While a batch runs, its child per-row buttons are disabled/greyed (spinner on the active control, plus an in-flight guard) to prevent double-submit; on completion the report re-lints once and surfaces partial failure (some pages may still fail grounding, and reappear under C3).
- **Remediation is durable across client disconnect; feedback is honest, not alarmist.** `POST /ingest` is a *synchronous* endpoint run in the server threadpool, so a client refresh/disconnect does **not** cancel it — the fix completes server-side (write under `_index_lock` via `wiki_writer`, idempotent by hash-skip), and only the HTTP response and the client's in-memory lint view are lost (recovered by re-running Lint). The UI therefore shows a **soft** notice ("runs in the background; if you leave, re-run Lint to see the result") plus a `beforeunload` confirm while a request is in-flight — never a false "don't refresh or you'll corrupt it," because that is untrue. Progress is an **indeterminate spinner + a source-count time estimate**; a true per-source "N/M done" bar is possible only by moving batch re-ingest onto the async `aingest_sources` submit/poll path (deferred, #247), and an intra-source % is not achievable (LLM calls are opaque). Concurrency cannot corrupt state — `_index_lock` serialises all ingest/lint, so overlapping triggers at worst repeat a forced re-ingest (redundant LLM cost) or no-op a hash-skip; the UI's disable-on-click + in-flight guard exist to save cost and confusion, the lock to guarantee integrity.
- **Two findings need new lifecycle endpoints, deferred to tier B.** C11 orphan-delete has no entity/concept delete endpoint and would **reopen [[adr-0012]]**'s deliberate "no general page delete" narrowing; C9 stale-qa's documented "re-ingest the entity" remediation does not actually refresh the *live* qa page (re-ingesting the source page leaves the Filed Answer stale), so it needs a demote-then-re-file operation that does not yet exist. Both are decided on their own, not smuggled into tier A.
- **Coherence remediation is grounding-satisfiable.** Because C5/C4 pages already have Sources, an LLM-drafted merged/reconciled page can be grounded in the *union* of the source pages' Sources and pass [[grounding-check]] — making C5/C4 Reconcile the cleanest tier-B showcase, ahead of Coverage (which depends on Import).
- **One taxonomy, three interfaces.** The four [[lint-axis]]es, the canonical check labels (C6 = stale, C3 = failed-grounding, …), and their zh/en strings live in a single domain taxonomy consumed by the Browser card, the CLI text output, and the MCP text output, so axis grouping and labelling are guaranteed identical across surfaces (ADR-0017) and i18n has one home.
- **C9 already covers concept pages, not only entities.** The C9 check iterates `entities/` *and* `concepts/`; the `entity_lookup` variable name and "entity" wording in its docstring are loose terminology, not a coverage gap. (Cleanup, not a design change.)
- **Governance carries over unchanged.** The read-only lint invariant (PRD #65 Q3 / #78) and the validated-write-back acceptance layer ([[adr-0020]]) both extend to remediation without modification — this ADR composes them, it does not revise them.
- **Invariants (tagged for the reviewer, per CODING_STANDARD §2.5).** **Invariant** — lint stays read-only: a Remediation is always a separate operation triggered from the report, never a side-effect of `run_lint()`. **Invariant** — batch remediation is offered on Direct-tier findings only (never Coherence/Coverage), preserving the per-item human-approval gate. **Invariant** — a C3 failed-grounding re-ingest (single or batch) MUST send `force:true`, or hash-skip (#93) no-ops it into a false fix. **Invariant** — no Authored Remediation bypasses grounding + human approval ([[adr-0020]]).
- **Implementation** is follow-up slices (tier A first); this ADR records the design from the grill.
