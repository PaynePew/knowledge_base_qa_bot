# The public `ask-wiki-rag` tenant is a deliberately ephemeral demo — no persistence, periodic reset to seed — and the write/curate lifecycle is made *legible* rather than hidden

`ask-wiki-rag` is a fully-open (no-auth), cost-capped (≤ $15/mo, $3/day) public demo behind the shared `edge` Caddy. It has **no named volume**: `deploy.yml` recreates the container on every push to `main`, and `reset.yml` runs `up -d --force-recreate` on a ~2-day cron, so all runtime writes (ingests, Filed Answers, promotions, index rebuilds) are **wiped to the committed seed** — deliberately ([`project-docs/deploy/README.md`](../deploy/README.md): "baked, read-only, ephemeral … no named volume and no backup"). The Filed Answer → Promote → Curate loop and the rest of the write/maintain lifecycle are designed for a **single local operator with a persistent disk** ([[operator-console]]); the public demo exposes the same UI, but its writes are transient by design. A 2026-06-30 grill (issues #341 item 1, #342) confirmed the "prod feels fragile" reports were this ephemeral wipe surfacing as apparent bugs (a promoted answer reverting to draft after a redeploy/reset). This ADR records the posture decision: **do not persist the public demo; make its ephemerality and its lifecycle steps legible instead.**

## Considered Options

- **Persist runtime writes via a named volume over `/app/wiki` + `/app/.kb` (rejected).** Would make promotions/ingests survive redeploy. Rejected because persisting a **fully-open, no-auth** endpoint means unbounded and potentially abusive content accumulation, which breaks the ≤ $15/mo cost cap and defeats the deliberate reset-to-seed hygiene that keeps the shared box clean. The curate loop's home is the persistent **local operator** environment, not the public demo — persistence is the right default *there*, not here.
- **Disable or hide the write actions in the public demo (rejected as the default; kept as a fallback).** Would remove the confusion at the cost of gutting the showcase — the whole point of the public demo is to let a visitor (e.g. an interviewer) run the full Import → Ingest → Index → Lint → Curate → Promote lifecycle live. We instead keep the actions functional and make their transience legible. If anonymous-write cost/abuse ever becomes a real problem, the fallback is to make the public console view-only (mutations local-only) while keeping the reader open.

## Consequences

- **No named volume; `reset.yml` retained** as the intentional cost/abuse-hygiene reset. The ephemeral posture stays exactly as `deploy/README.md` documents it.
- **Legibility, not removal.** Two demo-facing additions make the posture self-evident:
  1. A persistent banner on the [[operator-console]] stating the environment is a demo whose changes are real but ephemeral — it resets to the seeded knowledge base periodically (~2 days) and on each deploy.
  2. A concise, always-visible explanation beside each lifecycle action (Upload / Import / Ingest / Index / Lint / Promote / Discard / RAG rebuild) — *what the step does* and *its principle* (e.g. "Ingest: an LLM synthesises a curated wiki page from a Source, grounding-checked"; "Promote: a curator approves a draft Filed Answer so it becomes retrievable — the human gate of [[adr-0020]]'s validated write-back"). Always-visible one-liners (not hover-only tooltips) so the demo is legible at a glance; longer "how it works" copy may be an expandable. English, matching the product UI.
- **The demo teaches the architecture.** Beyond fixing the "feels fragile" perception, the per-step copy turns the console into a self-explaining walkthrough of the system's design — the intended audience is first-time visitors and interviewers.
- **Orthogonal to the FM-1 logic fix.** [[adr-0020]] makes the Filed Answer loop *correct* in the persistent local environment; this ADR decides that the *public demo* deliberately does not persist and instead explains itself. The two together close out the "prod feels fragile" review (#341/#342).
- **Implementation** (banner + per-step copy) is a follow-up frontend slice on `gateway/static/console.html`.
