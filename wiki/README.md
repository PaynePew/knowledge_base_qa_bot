# `wiki/` — generated artifacts and curated pages

This folder holds three classes of files. Only **compounding artifact** files are committed.

## Artifact taxonomy

| Class | Example | In git? | Editable by hand? | Produced by |
|---|---|---|---|---|
| Generated index | `index.md`, `lint-report.md` | ❌ (gitignored) | ❌ | `POST /index`, `POST /lint` |
| Runtime trace | `log.md` | ❌ (gitignored) | ❌ | server runtime (`logger.log_event`) |
| Working-memory cache | `hot.md` | ❌ (gitignored) | ✅ | `kb_save_hot_v1` MCP tool (Phase 12) |
| Compounding artifact | `entities/*.md`, `concepts/*.md` | ✅ | ❌ regenerated on re-ingest; edit the Source instead | `POST /ingest` |

**Compounding artifact** follows the same pattern as `package-lock.json`: committed to git for shareability, audit, and reproducibility, but regenerated by tooling (the `/ingest` pipeline), not hand-edited. The Source (`docs/*.md`) is the ground truth; the compounding artifact is the LLM's compiled view. Editing a wiki page directly is equivalent to editing `package-lock.json` by hand — the next `POST /ingest` will overwrite your changes.

## `wiki/index.md` — what it looks like

Auto-generated after every `POST /index` call. Illustrative output for a 3-Source KB:

```markdown
<!-- Auto-generated by POST /index — manual edits will be overwritten. Regenerate by re-running /index. -->

# Wiki Index

> 3 Sources, 9 Sections indexed.

## account_help.md

- [Change Email Address](../docs/account_help.md#change-email-address) — `account_help.md#change-email-address`
- [Reset Password](../docs/account_help.md#reset-password) — `account_help.md#reset-password`
...
```

For the authoritative format spec, see [`markdown_kb/tests/wiki_index/test_project_wiki_index.py`](../markdown_kb/tests/wiki_index/test_project_wiki_index.py) (`EXPECTED` literal).

## Why is `index.md` gitignored?

It is a deterministic projection of `.kb/index.json`. Two clones of this repo with the same `docs/` content produce byte-identical `wiki/index.md`. Committing it would create merge-conflict noise without adding shared truth. The pattern matches `.kb/index.json` — also a runtime artifact, also gitignored.

## `wiki/hot.md` — working-memory hot cache (Phase 12)

`wiki/hot.md` is a per-user runtime file written by the MCP tool `kb_save_hot_v1`
(Phase 12, ADR-0016). It holds the agent's working-memory summary from the
previous session so the agent can recover context on startup by calling
`kb_read_hot_v1`.

**Key properties:**

- **Gitignored** — it is per-user state, not shared KB content. Each user's
  `wiki/hot.md` reflects *their* session history. Committing it would create
  noisy diffs and expose ephemeral agent reasoning to other users.
- **Excluded from the Section Index corpus** — `hot.md` is never ingested by
  `POST /ingest` and is never fed into the BM25 or RAG index. KB search results
  do not include its content. This invariant is maintained by the ingestion
  pipeline's `WIKI_DIR` glob (which excludes generated files like `hot.md`).
- **Written atomically** — `kb_save_hot_v1` uses a tmp-file + `os.replace`
  pattern (Windows-safe) so a crash mid-write never leaves a partial file.
- **First-session absence is normal** — if `hot.md` does not exist,
  `kb_read_hot_v1` returns `{"content": ""}` (empty SUCCESS), not an error.
  Create it by calling `kb_save_hot_v1` at the end of your first session.

See `wiki/hot.md.example` for the expected format. Copy or rename it to
`wiki/hot.md` if you want to pre-seed the cache manually.

## How compounding artifacts are produced

`POST /ingest` (see [ADR-0003](../project-docs/adr/0003-w2-layered-wiki-target-claude-obsidian.md)) writes synthesis pages under `entities/` and `concepts/`. These pages:

- Carry 7-field YAML frontmatter (`id`, `type`, `created`, `updated`, `sources`, `status`, `open_questions`)
- Are committed to git as the knowledge owner's curated artifact
- Pass through the Grounding Check verifier (`CitableContent` Protocol — see [ADR-0004](../project-docs/adr/0004-post-llm-grounding-check.md))
