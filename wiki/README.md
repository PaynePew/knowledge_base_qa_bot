# `wiki/` ÔÇö generated artifacts and curated pages

This folder holds three classes of files. Only **compounding artifact** files are committed.

## Artifact taxonomy

| Class | Example | In git? | Editable by hand? | Produced by |
|---|---|---|---|---|
| Generated index | `index.md`, `lint-report.md` | ÔØî (gitignored) | ÔØî | `POST /index`, `POST /lint` |
| Runtime trace | `log.md` | ÔØî (gitignored) | ÔØî | server runtime (`logger.log_event`) |
| Compounding artifact | `entities/*.md`, `concepts/*.md` | Ô£à | ÔØî regenerated on re-ingest; edit the Source instead | `POST /ingest` |

**Compounding artifact** follows the same pattern as `package-lock.json`: committed to git for shareability, audit, and reproducibility, but regenerated by tooling (the `/ingest` pipeline), not hand-edited. The Source (`docs/*.md`) is the ground truth; the compounding artifact is the LLM's compiled view. Editing a wiki page directly is equivalent to editing `package-lock.json` by hand ÔÇö the next `POST /ingest` will overwrite your changes.

## `wiki/index.md` ÔÇö what it looks like

Auto-generated after every `POST /index` call. Illustrative output for a 3-Source KB:

```markdown
<!-- Auto-generated by POST /index ÔÇö manual edits will be overwritten. Regenerate by re-running /index. -->

# Wiki Index

> 3 Sources, 9 Sections indexed.

## account_help.md

- [Change Email Address](../docs/account_help.md#change-email-address) ÔÇö `account_help.md#change-email-address`
- [Reset Password](../docs/account_help.md#reset-password) ÔÇö `account_help.md#reset-password`
...
```

For the authoritative format spec, see [`markdown_kb/tests/wiki_index/test_project_wiki_index.py`](../markdown_kb/tests/wiki_index/test_project_wiki_index.py) (`EXPECTED` literal).

## Why is `index.md` gitignored?

It is a deterministic projection of `.kb/index.json`. Two clones of this repo with the same `docs/` content produce byte-identical `wiki/index.md`. Committing it would create merge-conflict noise without adding shared truth. The pattern matches `.kb/index.json` ÔÇö also a runtime artifact, also gitignored.

## How compounding artifacts are produced

`POST /ingest` (see [ADR-0003](../project-docs/adr/0003-w2-layered-wiki-target-claude-obsidian.md)) writes synthesis pages under `entities/` and `concepts/`. These pages:

- Carry 7-field YAML frontmatter (`id`, `type`, `created`, `updated`, `sources`, `status`, `open_questions`)
- Are committed to git as the knowledge owner's curated artifact
- Pass through the Grounding Check verifier (`CitableContent` Protocol ÔÇö see [ADR-0004](../project-docs/adr/0004-post-llm-grounding-check.md))
